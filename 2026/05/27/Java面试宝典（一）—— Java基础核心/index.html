Java面试宝典

Java面试宝典（一）—— Java基础核心

引言

Java 基础是面试中必考的内容，无论面试官问得多深入，最终都会回到基础原理上来。本文将从最常考的知识点出发，系统梳理 Java 基础核心内容。

一、集合框架

1.1 HashMap 底层原理

面试官常问：请说一下 HashMap 的底层实现。

HashMap 的底层是数组 + 链表 + 红黑树（JDK 8+）。

  • JDK 8 之前：数组 + 链表，使用拉链法解决哈希冲突
  • JDK 8 之后：当链表长度超过 8 且数组长度 >= 64 时，链表转为红黑树，将查找时间复杂度从 O(n) 降为 O(log n)

核心参数：

参数 默认值 说明
DEFAULT_INITIAL_CAPACITY 16 初始容量
DEFAULT_LOAD_FACTOR 0.75 负载因子，扩容阈值 = 容量 × 负载因子
TREEIFY_THRESHOLD 8 链表转红黑树的阈值
UNTREEIFY_THRESHOLD 6 红黑树转链表的阈值

put 操作流程：

  1. 计算 key 的 hash 值（高 16 位与低 16 位异或，减少哈希冲突）
  2. 如果数组为空，先初始化
  3. 根据 hash 值定位到数组下标，如果该位置为空，直接插入
  4. 如果不为空，判断 key 是否相等，相等则覆盖 value
  5. 如果是红黑树节点，按树的方式插入
  6. 如果是链表节点，遍历插入到尾部
  7. 插入后检查链表长度，超过 8 则转为红黑树
  8. 检查是否需要扩容

扩容机制：

  • 当元素数量 > 容量 × 负载因子时触发扩容
  • 扩容为原来的 2 倍
  • JDK 8 优化了 rehash 过程，不需要重新计算 hash，只需判断原 hash 值新增的最高位是 0 还是 1，0 则位置不变，1 则位置变为”原位置 + 旧容量”

1.2 HashMap 和 Hashtable 的区别

特性 HashMap Hashtable
线程安全 否 是（方法级 synchronized）
允许 null 键/值 允许 不允许
性能 高 低（全表锁）
继承关系 AbstractMap Dictionary（已过时）

1.3 ConcurrentHashMap

面试官常问：ConcurrentHashMap 如何保证线程安全？

  • JDK 7：使用 Segment 分段锁，将数据分成多个 Segment，每个 Segment 独立加锁，锁粒度降低
  • JDK 8：放弃 Segment，采用 CAS + synchronized 保证线程安全。锁粒度进一步细化到数组的每个桶（链表头/树根节点），并发度更高

核心操作：

  • put：CAS 插入，如果桶为空则 CAS 写入，不为空则 synchronized 锁住桶头节点
  • get：无锁操作，通过 volatile 保证可见性
  • 扩容：支持多线程协助扩容，提高吞吐量

1.4 ArrayList 和 LinkedList

特性 ArrayList LinkedList
底层结构 动态数组 双向链表
随机访问 O(1) O(n)
头/中间插入删除 O(n)（需要移动元素） O(1)（需先定位 O(n)）
尾部插入 O(1)（均摊） O(1)
内存占用 少（仅数组 + size） 多（每个节点有 prev/next 指针）

ArrayList 扩容： 默认初始容量 10，扩容为原来的 1.5 倍，使用 Arrays.copyOf 进行数组复制。

1.5 fail-fast 和 fail-safe

  • fail-fast：HashMap、ArrayList 等在遍历过程中如果被结构性修改，会抛出 ConcurrentModificationException。通过 modCount 变量检测
  • fail-safe：ConcurrentHashMap、CopyOnWriteArrayList 等基于数据副本或弱一致性迭代器，不会抛异常

二、并发编程

2.1 线程的创建方式

  1. 继承 Thread 类
  2. 实现 Runnable 接口（推荐，避免单继承限制）
  3. 实现 Callable 接口（有返回值，可抛异常）
  4. 线程池创建（生产环境推荐）

2.2 线程状态

  • NEW：新建，未调用 start()
  • RUNNABLE：可运行，包括运行中 + 等待 CPU 时间片
  • BLOCKED：等待锁
  • WAITING：无限期等待，需要被唤醒（Object.wait、Thread.join）
  • TIMED_WAITING：限期等待（Thread.sleep、wait(timeout)）
  • TERMINATED：线程结束

2.3 synchronized 和 ReentrantLock

特性 synchronized ReentrantLock
实现层面 JVM 层面（monitor） JDK 层面（AQS）
锁释放 自动释放 需手动 unlock()
公平锁 不支持 支持
可中断 不支持 支持（lockInterruptibly）
条件变量 单一（wait/notify） 多个 Condition
锁尝试 不支持 tryLock()

synchronized 的锁升级：

  • 偏向锁：多数情况锁由同一线程多次获得，通过 Mark Word 记录线程 ID
  • 轻量级锁：线程竞争时，使用 CAS 自旋，不阻塞
  • 重量级锁：自旋超过一定次数，升级为重量级锁，线程挂起等待

2.4 volatile 关键字

作用：

  1. 保证可见性：修改后立即写回主存，其他线程立即可见
  2. 禁止指令重排序：通过内存屏障实现

不保证原子性：volatile int i; i++ 不是原子操作（读-改-写三步）

底层原理：通过 lock 前缀指令触发 Lock 缓存行，保证多核 CPU 间的缓存一致性（MESI 协议）。

2.5 AQS（AbstractQueuedSynchronizer）

面试官常问：请简述 AQS 的原理。

AQS 是 JDK 并发包的核心框架，ReentrantLock、Semaphore、CountDownLatch 等都基于它。

核心思想：

  • 使用一个 volatile int state 变量表示同步状态
  • 使用 FIFO 双向队列（CLH 变种）管理等待线程
  • 子类通过重写 tryAcquire / tryRelease 实现具体的同步逻辑

获取锁流程：

  1. 尝试 tryAcquire（CAS 修改 state）
  2. 失败则封装为 Node 加入等待队列尾部
  3. 挂起当前线程（LockSupport.park）
  4. 被唤醒后重新尝试获取

7 个核心参数：

参数 说明

unit keepAliveTime 单位
workQueue 任务队列
threadFactory 线程工厂

执行流程：

拒绝策略：

  • AbortPolicy：抛出异常（默认）
  • CallerRunsPolicy：由调用线程执行
  • DiscardPolicy：直接丢弃
  • DiscardOldestPolicy：丢弃队列最老任务

生产环境不建议使用 Executors 创建线程池（FixedThreadPool 和 SingleThreadPool 队列无界可能 OOM，CachedThreadPool 最大线程数无界可能 OOM）。

2.7 ThreadLocal

  • 每个线程持有独立的变量副本，通过 Thread.threadLocals 存储
  • 底层是 ThreadLocalMap，key 是 ThreadLocal 的弱引用
  • 内存泄漏问题：key 被 GC 回收后 value 不会被自动清理，需要手动 remove()

三、JVM

3.1 内存结构

堆内存划分（JDK 8）：

  • 新生代：Eden（8/10）+ Survivor0（1/10）+ Survivor1（1/10）
  • 老年代：经过多次 GC 仍然存活的对象

3.2 GC 算法

  • 标记-清除：产生内存碎片
  • 标记-复制：新生代使用，复制存活对象到 Survivor 区
  • 标记-整理：老年代使用，将存活对象向一端移动

3.3 垃圾收集器

收集器 区域 算法 特点
Serial 新生代 标记-复制 单线程，Client 模式默认
Parallel Scavenge 新生代 标记-复制 多线程，关注吞吐量
CMS 老年代 标记-清除 低停顿，会产生碎片
G1 全堆 标记-复制+整理 可预测停顿时间，JDK 9+ 默认
ZGC 全堆 着色指针+读屏障 亚毫秒级停顿，JDK 15+ 正式

对象是否存活的判断：

  • 引用计数法：循环引用无法回收（Python 使用，JVM 不用）
  • 可达性分析：从 GC Roots 出发，不可达的对象将被回收

GC Roots 包括：

  • 虚拟机栈中的局部变量
  • 方法区中静态属性
  • 方法区中常量
  • 本地方法栈中的 JNI 引用

双亲委派模型：

子类加载器收到请求时，先委托父类加载，父类无法完成时才自己加载。好处：防止核心类被篡改，保证类的唯一性。

五、常见面试题

Q1：== 和 equals 的区别？

  • ==：比较基本类型的值，比较引用类型的地址
  • equals：默认行为是 ==（Object 类），可被重写为比较内容（如 String）

Q2：String、StringBuilder、StringBuffer 的区别？

类 可变性 线程安全 性能
String 不可变 是 -
StringBuffer 可变 是（synchronized） 中
StringBuilder 可变 否 高

Q3：接口和抽象类的区别？

特性 接口 抽象类
继承 多实现 单继承
成员变量 只能是常量 可以有各种变量
构造方法 无 有
JDK 8+ 可以有 default/static 方法 无变化

Q4：泛型擦除是什么？

泛型只在编译期有效，编译后泛型信息被擦除为原始类型（raw type）。List<String> 和 List<Integer> 在运行时都是 List。

Q5：反射的性能开销？

  • 方法调用比直接调用慢，因为需要类型检查、安全检查等
  • 优化：setAccessible(true) 跳过访问检查
  • 现代 JVM 对反射有内联优化，差距在缩小

总结

Java 基础是面试的基石。集合框架重点掌握 HashMap 和 ConcurrentHashMap，并发编程重点掌握 AQS 和线程池，JVM 重点掌握 GC 和类加载机制。理解原理而非死记硬背，才能在面试中灵活应对。
