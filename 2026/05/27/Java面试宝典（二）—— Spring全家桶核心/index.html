Java面试宝典（二）—— Spring全家桶核心

引言

Spring 全家桶是 Java 后端开发的事实标准。面试中，Spring 相关问题几乎必考。本文将从 Spring Framework 到 Spring Boot 再到 Spring Cloud，梳理最常考的核心知识点。

一、Spring Framework

1.1 IoC（控制反转）

面试官常问：什么是 IoC？Spring 的 IoC 是怎么实现的？

IoC（Inversion of Control）是将对象的创建和依赖关系的维护交给容器，而非由开发者手动 new 对象。

实现原理：

  • 反射：Spring 通过反射机制动态创建 Bean 实例
  • XML/注解/Java Config：定义 Bean 的元信息
  • BeanFactory / ApplicationContext：核心容器，负责 Bean 的生命周期

Bean 的生命周期：

Bean 的作用域：

作用域 说明
singleton 单例，默认（容器级别唯一）
prototype 多例，每次获取创建新实例
request 每次 HTTP 请求创建新实例
session 每个 HTTP Session 唯一

1.2 DI（依赖注入）

IoC 的具体实现方式。Spring 支持三种注入方式：

  • 构造器注入（推荐）：依赖不可变，适合强制依赖，方便单元测试
  • Setter 注入：适合可选依赖
  • 字段注入（@Autowired，不推荐）：隐藏依赖关系，不利于测试

1.3 AOP（面向切面编程）

面试官常问：Spring AOP 的原理是什么？

AOP 将横切关注点（日志、事务、安全等）从业务逻辑中分离出来。

底层实现：

  • JDK 动态代理：目标类实现了接口时使用。通过 Proxy.newProxyInstance 创建代理
  • CGLIB 代理：目标类没有接口时使用。通过字节码生成目标类的子类，重写方法

核心概念：

概念 说明
切面（Aspect） 横切关注点的模块化（如日志切面）
连接点（JoinPoint） 程序执行过程中的一个点（方法调用）
通知（Advice） 在连接点执行的动作
切点（Pointcut） 匹配连接点的表达式
织入（Weaving） 将切面应用到目标对象的过程

1.4 Spring 事务

面试官常问：Spring 事务的传播行为有哪些？

7 种传播行为：

传播行为 说明
REQUIRED 默认。当前有事务则加入，没有则新建
REQUIRES_NEW 挂起当前事务，创建新事务
SUPPORTS 有事务则加入，没有则以非事务方式执行
NOT_SUPPORTED 以非事务方式执行，挂起当前事务
MANDATORY 必须在事务中，否则抛异常
NEVER 必须不在事务中，否则抛异常
NESTED 嵌套事务（Savepoint）

事务失效的常见场景：

  1. 方法非 public：Spring AOP 只拦截 public 方法
  2. 方法自调用：同一类内部方法调用不经过代理
  3. 异常被 catch：事务切面捕获不到异常，不会回滚
  4. 异常类型不匹配：默认只回滚 RuntimeException 和 Error
  5. 多线程：子线程不在同一事务上下文中

解决自调用失效：

  • 注入自己（@Autowired private XxxService self;）
  • 使用 AopContext.currentProxy()
  • 将方法拆分到不同类中

三、Spring Boot

3.1 自动配置原理

面试官常问：Spring Boot 自动装配是怎么实现的？

核心注解：@SpringBootApplication = @Configuration + @EnableAutoConfiguration + @ComponentScan

自动配置流程：

  1. @EnableAutoConfiguration 导入 AutoConfigurationImportSelector
  2. 读取 META-INF/spring.factories（Spring Boot 2.x）或 META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports（Spring Boot 3.x）
  3. 根据 @Conditional 系列注解按需加载配置

常用 @Conditional 注解：

注解 条件
@ConditionalOnClass classpath 中存在指定类
@ConditionalOnMissingBean 容器中不存在指定 Bean
@ConditionalOnProperty 配置文件中存在指定属性
@ConditionalOnWebApplication 是 Web 应用

3.2 Starter 机制

Starter 是一组依赖的聚合，开发者只需引入一个 Starter，就能获得完整的功能支持。

3.3 配置文件加载顺序

Spring Boot 支持 application.properties 和 application.yml，加载优先级从高到低：

  1. 命令行参数
  2. SPRING_APPLICATION_JSON 环境变量
  3. application-{profile}.properties/yml（外部）
  4. application.properties/yml（外部）
  5. @PropertySource 注解
  6. application-{profile}.properties/yml（classpath）
  7. application.properties/yml（classpath）

3.4 Spring Boot 3.x 的变化

  • 最低要求 JDK 17
  • Jakarta EE 10（javax.* → jakarta.*）
  • 支持 GraalVM 原生编译
  • 引入 AOT 编译优化

4.5 熔断与限流（Sentinel）

面试官常问：熔断、降级、限流的区别是什么？

  • 限流：限制请求的 QPS 或并发数，超过阈值直接拒绝（防止系统被打垮）
  • 熔断：下游服务异常率升高时，快速失败，防止雪崩（类似电路熔断器）
  • 降级：服务不可用时，返回兜底数据（如缓存、默认值）

Sentinel 三种流控模式：

模式 说明
直接模式 资源超过阈值直接拒绝
关联模式 关联资源达到阈值时，限流当前资源
链路模式 从指定入口进来的请求才限流

五、常见面试题

Q1：Spring 循环依赖怎么解决的？

Spring 通过三级缓存解决单例循环依赖：

  1. singletonObjects（一级）：成品 Bean，可直接使用
  2. earlySingletonObjects（二级）：半成品 Bean，已实例化但未完成属性注入
  3. singletonFactories（三级）：ObjectFactory，用于创建早期引用

注意：构造器注入的循环依赖无法解决，会抛出 BeanCurrentlyInCreationException。

Q2：Spring Bean 的线程安全性？

  • Spring Bean 默认单例，有状态的 Bean 不是线程安全的
  • 解决方案：
    • 尽量将 Bean 设计为无状态的
    • 使用 ThreadLocal 保存线程私有变量
    • 将 Bean 作用域改为 prototype

Q3：Spring Boot 内置 Tomcat 是怎么启动的？

  • spring-boot-starter-web 引入了 Tomcat 依赖
  • ServletWebServerApplicationContext 在 refresh() 时调用 createWebServer()
  • 通过 TomcatServletWebServerFactory 创建并启动 Tomcat

Q4：Spring 事务嵌套会发生什么？

取决于传播行为：

  • REQUIRED：外层事务回滚，内层也会回滚（同一事务）
  • REQUIRES_NEW：内层是独立事务，外层回滚不影响内层

Q5：微服务之间怎么保证数据一致性？

  • 最终一致性：基于消息队列（RocketMQ 事务消息）
  • Saga 模式：每个操作有对应的补偿操作
  • TCC 模式：Try-Confirm-Cancel，强一致但复杂度高
  • 本地消息表：业务操作和消息写入在同一本地事务

总结

Spring 全家桶的核心在于理解 IoC/AOP 的设计思想，掌握 Spring Boot 的自动配置原理，以及了解 Spring Cloud 各组件的职责和协作方式。面试中不必死记源码，但要能说清楚关键流程的设计意图。
